# 招标技术文件自动审核系统 - 使用说明书

## 1. 系统简介

本系统用于自动审核招标技术文件与标准化技术文件模板的差异，支持技术参数对比、设备选型审核、技术条款变更审核和合规性检查，帮助用户快速发现招标文件中的问题和风险。

### 1.1 主要功能

- **模板管理**: 上传和管理标准化技术文件模板
- **项目管理**: 上传项目招标技术文件并关联模板
- **智能审核**: 基于AI自动对比分析文件差异
- **风险评估**: 对差异进行高/中/低风险等级划分
- **审核报告**: 生成详细的审核报告并支持导出

### 1.2 审核范围

| 审核类型 | 说明 |
|---------|------|
| 技术参数差异 | 检查参数值是否超出模板范围 |
| 设备选型变更 | 检查设备型号、品牌是否变更 |
| 技术条款变更 | 检查技术条款是否修改、新增或删除 |
| 合规性问题 | 检查是否符合GB国家标准、DL/T电力行标等 |

## 2. 快速开始

### 2.1 系统要求

- Docker Desktop 已安装并运行
- 现代浏览器（Chrome、Edge、Firefox等）

### 2.2 启动系统

```bash
# 进入项目目录
cd bid-audit-system

# 启动服务
docker compose up -d

# 查看服务状态
docker compose ps
```

### 2.3 访问系统

打开浏览器访问：**http://localhost**

## 3. 用户管理

### 3.1 用户注册

1. 访问系统登录页面
2. 点击"注册"按钮
3. 填写用户名、密码
4. 选择角色（管理员/审核员/查看者）
5. 点击"确认注册"

### 3.2 用户登录

1. 在登录页面输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后自动跳转到工作台

### 3.3 角色权限

| 角色 | 权限说明 |
|------|---------|
| 管理员 | 所有功能权限，包括用户管理、规则管理 |
| 审核员 | 上传模板、创建项目、执行审核 |
| 查看者 | 查看审核结果和报告 |

## 4. 功能使用指南

### 4.1 工作台

登录后首页面板，显示：
- 模板数量统计
- 项目数量统计
- 审核次数统计
- 最近审核记录列表

### 4.2 模板管理

#### 上传模板

1. 进入"模板管理"页面
2. 点击"上传模板"按钮
3. 填写模板信息：
   - **模板名称**: 如"变电站技术规范模板"
   - **描述**: 模板用途说明
   - **版本**: 如"1.0"
   - **模板文件**: 选择 `.docx` 格式的Word文件
4. 点击"上传"完成

#### 管理模板

- 查看模板列表：显示所有已上传的模板
- 删除模板：点击"删除"按钮（仅管理员可操作）

### 4.3 项目管理

#### 创建项目

1. 进入"项目管理"页面
2. 点击"创建项目"按钮
3. 填写项目信息：
   - **项目名称**: 如"XX变电站招标文件"
   - **描述**: 项目说明
   - **关联模板**: 选择对应的标准化模板
   - **项目文件**: 选择 `.docx` 格式的招标文件
4. 点击"创建"完成

#### 管理项目

- 查看项目列表及状态（待审核/已审核）
- 直接跳转到审核页面
- 删除项目

### 4.4 文件审核

#### 执行审核

1. 进入"文件审核"页面
2. 选择要审核的项目
3. 点击"开始审核"按钮
4. 等待AI分析完成（通常需要1-3分钟）

#### 审核过程

系统会自动执行以下分析：
1. 解析模板文件和项目文件
2. 提取技术参数、条款、表格等内容
3. 对比分析差异
4. 评估风险等级
5. 生成审核报告

### 4.5 审核结果

审核完成后，页面显示：

#### 审核概要

- 审核ID
- 审核状态
- 总体风险等级（高/中/低）
- 差异总数及分布

#### 详细差异清单

| 字段 | 说明 |
|------|------|
| 类型 | 参数差异/设备变更/条款变更/合规问题 |
| 类别 | 差异分类 |
| 位置 | 在文档中的位置 |
| 风险等级 | 高风险/中风险/低风险 |
| 描述 | 差异详细说明 |
| 建议 | 修改建议 |

#### 导出报告

点击"导出报告"按钮，可下载审核报告文件。

## 5. 配置说明

### 5.1 环境变量配置

配置文件位置：`backend/.env`

```env
# 数据库配置
DATABASE_URL=sqlite+aiosqlite:///./audit.db

# 安全密钥
SECRET_KEY=your-secret-key

# LLM配置
OPENAI_API_KEY=your-api-key
OPENAI_API_BASE=https://api.deepseek.com/v1
DEFAULT_LLM_MODEL=deepseek-chat

# 调试模式
DEBUG=true
```

### 5.2 支持的LLM模型

| 模型 | API Base | 说明 |
|------|----------|------|
| DeepSeek | https://api.deepseek.com/v1 | 推荐，国内访问快 |
| OpenAI GPT | https://api.openai.com/v1 | 需要科学上网 |
| Claude | - | 需要配置ANTHROPIC_API_KEY |

### 5.3 修改配置后重启服务

```bash
docker compose down
docker compose up -d
```

## 6. 常见问题

### Q1: 上传文件失败

**原因**: 文件格式不支持或文件过大

**解决**: 
- 确保文件为 `.docx` 格式
- 检查文件大小不超过50MB

### Q2: 审核超时

**原因**: 文件内容过多或API调用失败

**解决**:
- 检查API Key是否正确配置
- 检查网络连接是否正常
- 尝试减小文件内容

### Q3: 无法登录

**原因**: 用户名密码错误或用户不存在

**解决**:
- 确认用户名和密码正确
- 联系管理员重置密码

### Q4: Docker服务启动失败

**解决**:
```bash
# 查看日志
docker compose logs

# 重新构建并启动
docker compose down
docker compose build --no-cache
docker compose up -d
```
